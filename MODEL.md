# Model

SkySight использует YOLO-модель через Ultralytics для обнаружения объектов на кадрах с камеры БПЛА или симулятора.

## где лежит модель

Путь по умолчанию:

```text
data/models/best_yolo11.pt
```

Он задаётся в `fire_uav/config/settings_default.json`.

## параметры

| параметр | смысл |
|---|---|
| `yolo_model` | путь к `.pt`-файлу модели |
| `yolo_conf` | минимальная уверенность детекции |
| `yolo_iou` | IoU-порог для NMS |
| `yolo_classes` | список классов, которые нужно оставить |

## замена модели

1. положить новый файл в `data/models/`;
2. изменить `yolo_model` в конфиге;
3. при необходимости изменить `yolo_classes`;
4. запустить проверку на тестовом видео или в Unreal-сцене;
5. проверить, что bbox совпадают с ожидаемыми классами.

## устройство инференса

Если CUDA доступна, модель запускается на GPU. Если CUDA недоступна, используется CPU.

Практически:

- для разработки можно использовать CPU;
- для демонстрации лучше GPU;
- для БПЛА/Jetson нужно отдельно проверить совместимость PyTorch/JetPack;
- для слабых устройств лучше использовать маленькие веса и более высокий `yolo_conf`.

## рекомендации по датасету

Для задачи SkySight желательно, чтобы в датасете были:

- огонь в лесной среде;
- дым с разной плотностью;
- люди в лесу и на открытой местности;
- кадры сверху и под углом, похожие на БПЛА;
- разные высоты полёта;
- дневное/вечернее освещение;
- сложные ложные объекты: тени, облака, костры, светлые камни, туман.

## метрики

| метрика | зачем нужна |
|---|---|
| precision | насколько мало ложных тревог |
| recall | насколько мало пропущенных объектов |
| mAP@0.5 | общее качество детекции |
| mAP@0.5:0.95 | более строгая оценка качества bbox |
| FPS | скорость обработки кадров |
| latency | задержка от кадра до события |

## экспорт

Для развёртывания можно рассмотреть экспорт в ONNX, TensorRT, OpenVINO или вынести инференс в отдельный внешний сервис.
