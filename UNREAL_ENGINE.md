# Unreal Engine

`unrealbridge/` содержит исходники симулятора SkySight на Unreal Engine 5. Симулятор нужен для безопасной отладки полёта, телеметрии, видеопотока, пожара, дыма и сценариев обнаружения без реального БПЛА.

## роль симулятора

Симулятор выполняет четыре задачи:

1. создаёт управляемый БПЛА в сцене;
2. отдаёт телеметрию Python-модулю;
3. отдаёт поток камеры;
4. принимает маршруты и команды управления.

## структура

```text
unrealbridge/
├── SkySight.uproject
├── SkySight.sln
├── Source/SkySight/
└── native_core/
```

## HTTP API

| метод | endpoint | назначение |
|---|---|---|
| `GET` | `/sim/v1/telemetry` | положение, ориентация, скорость, заряд, режим |
| `GET` | `/sim/v1/video.ts` | H.264 видеопоток с камеры |
| `POST` | `/sim/v1/route` | загрузка маршрута |
| `POST` | `/sim/v1/command` | команда управления |

## команды

| команда | смысл |
|---|---|
| `HOLD` / `PAUSE` | остановить выполнение маршрута |
| `RESUME` | продолжить маршрут |
| `RTL` | возврат |
| `SET_SPEED` | изменить скорость |
| `ORBIT_START` | начать облёт цели |
| `ORBIT_STOP` | выйти из облёта и вернуться к маршруту |
| `RESET` | сбросить состояние |

## режим без Unreal

Если симулятор не нужен или Unreal ещё не запущен, используйте стаб:

```bash
UNREAL_BRIDGE_PORT=9000 python scripts/unreal_bridge_stub.py
```

## что добавить в сцену для демонстрации

- лесной участок или полигон;
- несколько источников огня;
- дымовые эффекты с разной плотностью;
- зоны с людьми или силуэтами;
- no-fly зоны;
- стартовую позицию БПЛА;
- камеру БПЛА с понятным FOV;
- визуальные маркеры маршрута и целей.

## чек-лист отладки

- `/sim/v1/telemetry` отвечает JSON-ом;
- координаты меняются при движении дрона;
- `/sim/v1/video.ts` отдаёт поток;
- `POST /sim/v1/route` меняет маршрут;
- `POST /sim/v1/command` обрабатывает hold/resume/orbit;
- Python-адаптер не теряет соединение;
- частота телеметрии и видео соответствует настройкам.
